**开发计划：登月（Moon Lander）游戏**

这是一款经典的上世纪70年代游戏，通常在文本终端或简单的图形界面上运行。目标是控制一艘登月舱在月球表面安全着陆，同时管理燃料消耗。

**1. 核心游戏机制**

*   `x, y`: 位置（垂直和水平）
*   `vx, vy`: 速度（垂直和水平）
*   `fuel`: 剩余燃料
*   `angle`: 登月舱角度（用于控制推力方向）
*   **玩家输入:**
    *   控制推力（向上）
    *   控制左右姿态（旋转）
*   **物理模拟:**
    *   重力（月球重力）
    *   推力（根据燃料消耗和角度计算）
*   **胜利条件:**
    *   安全着陆：速度在允许范围内，角度在允许范围内，接触到月球表面。
*   **失败条件:**
    *   坠毁：速度过快或角度不当接触月球表面。
    *   燃料耗尽：无法控制登月舱。
    *   飞出屏幕：飞离游戏区域。
*   **月球表面:**
    *   随机生成不规则地形，包含平坦着陆区。

**2. 技术栈（考虑到可在卡西欧计算器上玩，我们将选择极简的Web技术）**

*   **HTML:** 游戏结构和画布。
*   **CSS:** 简单的样式，如背景、UI元素。
*   **JavaScript:** 核心游戏逻辑、物理模拟、渲染。
    *   不使用复杂的框架或库，直接操作DOM或Canvas。

**3. 开发步骤**

*   **阶段一：基础框架与物理模拟**
    *   **创建HTML结构:** `index.html`，包含一个 `<canvas>` 元素用于游戏渲染。
    *   **初始化游戏状态:** 定义登月舱的初始位置、速度、燃料等。
    *   **实现游戏循环:** 使用 `requestAnimationFrame`。
    *   **实现基本物理:**
        *   月球重力作用于 `vy`。
        *   根据 `vx, vy` 更新 `x, y`。
    *   **基本渲染:** 在Canvas上绘制一个简单的登月舱方块和月球表面线条。
    *   **玩家输入:** 实现键盘事件监听，用于控制登月舱的推力和旋转。
    *   **燃料消耗:** 推力操作消耗燃料。
    *   **碰撞检测:** 登月舱与月球表面的碰撞检测。
    *   **基本胜负判断:** 粗略的速度和角度判断。

*   **阶段二：游戏元素与用户界面**
    *   **月球地形生成:** 实现简单的随机月球地形生成算法，确保有平坦的着陆区域。
    *   **登月舱图形:** 绘制更具表现力的登月舱（例如，使用Canvas的 `beginPath`, `lineTo`, `fill` 等）。
    *   **火焰效果:** 登月舱推力时显示简单的火焰动画。
    *   **HUD (平视显示器):**
        *   显示当前速度（水平、垂直）。
        *   显示剩余燃料。
        *   显示登月舱角度。
        *   显示着陆区域距离（可选）。
    *   **游戏开始/结束画面:**
        *   显示“欢迎”界面，包含操作说明。
        *   显示“胜利”界面，显示着陆得分。
        *   显示“失败”界面，显示坠毁原因。
    *   **音效 (可选):** 简单的推力、爆炸、着陆音效。

*   **阶段三：优化与扩展**
    *   **分数系统:** 根据燃料、着陆精度等计算分数。
    *   **多关卡/难度:** 不同的地形、初始条件。
    *   **排行榜 (可选):** 本地存储最高分。
    *   **配置项:** 提取游戏中的数值（重力、燃料量、安全着陆速度/角度等）到 `config.js` 文件中，方便调整。

**4. 目录结构设想**

```
Super Mario/ (项目根目录)
├── index.html          // 游戏主页
├── style.css           // 样式文件
├── script.js           // 核心游戏逻辑
├── config.js           // 游戏配置项
├── assets/ (可选，如果需要图片或音效)
│   ├── images/
│   └── sounds/
└── changelog.md        // 变更日志 (保留用户已有的)
```

*   对核心物理模拟函数（如重力计算、推力计算）进行单元测试。
*   对碰撞检测逻辑进行测试。
*   对游戏状态更新函数进行测试。
